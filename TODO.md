# Description

1. **核心功能目标**
   - 支持用户自定义多维度检测规则（如指定群聊的特定话题、特定群友互动行为等）
   - LLM识别到匹配规则的内容后，通过邮件等方式向用户推送提醒
2. **基础规则设置能力**
   - 支持精准场景规则：如“A群聊到汽车时提醒、B群讨论黑苹果白屏问题时提醒”
   - 支持社交行为规则：如“C群中x群友和y群友出现互动/暧昧行为时提醒”
3. **技术架构与消息来源**
   - 采用nonebot作为消息接入层，兼容不同消息平台的消息采集
   - 提供WebUI可视化操作界面，降低用户使用门槛
4. **LLM检测执行逻辑**
   - 支持配置同时处理的规则数量上限
   - 消息触发检测时，自动抓取该消息的前文（如10条），拼接成完整上下文
   - 按批次将上下文+检测规则组合后请求LLM API判断（如先判断前2条规则，再判断后2条规则）
5. **用户行为记忆与智能推荐**
   - 识别用户自身发送的消息，提取用户参与的话题、互动群友等信息并存储
   - 基于历史记忆自动生成建议的检测规则
   - 针对现有规则，随时间推移生成优化改进方案
6. **规则迭代与反馈机制**
   - 支持用户对检测结果打分，系统结合评分生成规则改进建议（如补充细节要求）
   - 支持“一句话生成结构化规则”：自动提取目标群聊/私聊ID（支持模糊识别，如“聊游戏的群”“我最喜欢的群”）、相关人员、时间、话题关键词，生成完整规则描述
7. **扩展能力与自定义配置**
   - 提供API/MCP接口：一句话生成规则既可用内置智能体，也支持用户自定义外部系统提示词
   - 数据库层：支持自定义数据库方案（基于ORM框架）
   - 规则触发扩展：除固定通知外，可调用自定义外部API
   - 规则参数化：支持为规则设定结果参数（如汽车话题规则的“品牌、领域、价格”），触发时由AI填充参数并传递给外部API（如二手交易群自动统计物品价格、成色等数据）

# Decisions

- 平台：QQ（OneBot）优先，同时保持适配层可扩展。
- 部署：本地 + Docker。
- 模型：云端与本地模型并行支持（统一 Provider 抽象）。
- WebUI：MVP 范围（规则管理 + 日志 + 反馈）。
- 数据：SQLite + SQLAlchemy 起步，并预留 ORM 适配接口。
- 执行：规则批间并行，兼顾速度与吞吐。
- 规则生成：内置Agent与外部自定义提示词路径同期支持。
